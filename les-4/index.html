Key take-aways Semantische tags (Sander Leer)

  • Gebruik van deprecated HTML elementen (, , ;) zijn uit den boze omdat ze iets zeggen over de vormgeving en niet over de inhoud.
  • Gebruik waar mogelijk het juiste HTML element voor wat je wilt doen.
  • Namen van classes die bestaande HTML elementen beschrijven moet je als waarschuwing zien dat je waarschijnlijk een ander HTML element kunt gebruiken. Bijvoorbeeld:
    • <p class="heading-1"> moet waarschijnlijk zijn <h1>,
    • <span class="important"> moet waarschijnlijk zijn <strong>,
    • <div class="footer-content"> moet waarschijnlijk zijn <footer>.
    • Namen van classes beschrijven nooit de vormgeving maar altijd de intentie (of semantiek). Bijvoorbeeld:
    • <p class="red"> moet waarschijnlijk zijn <p class="warning">,
    • <aside class="right"> moet waarschijnlijk zijn
  • Gebruik van id’s voor vormgeving met CSS is in 99% van de gevallen niet noodzakelijk (als in

    en #one { … }).

  • Gebruik van !important in je CSS is in 99% van de gevallen niet noodzakelijk (als in p { color: green !important;}).
  • Zorg dat – naast valide HTML en CSS – ook je Sectioning content en Outline op orde is.
